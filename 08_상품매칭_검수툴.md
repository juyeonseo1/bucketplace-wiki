# 상품 매칭 검수툴

Google Apps Script(GAS) 기반의 웹 검수 툴. 두 플랫폼 간 상품 매칭의 정확성(일치/불일치)을 다수의 작업자가 분업하여 검수하고, 관리자가 진행 현황을 모니터링하고 결과를 다운로드할 수 있다.

---

## 기술 구조

| 항목 | 내용 |
|------|------|
| 런타임 | Google Apps Script (GAS) |
| 프런트엔드 | 단일 HTML 파일 (index.html) — GAS `doGet()`으로 서빙 |
| 데이터 저장 | Google Spreadsheet (시트별 역할 분리) |
| 파일 파싱 | SheetJS (xlsx 라이브러리, CDN) |

### 스프레드시트 구조

**메인 시트** (스크립트가 연결된 시트)

| 시트명 | 역할 |
|--------|------|
| Projects | 프로젝트 목록 (ID, 이름, 생성일, 스프레드시트ID, 모드, 스키마JSON) |
| Workers | 전역 작업자 풀 (모든 프로젝트 공유) |

**프로젝트별 시트** (프로젝트 생성 시 별도 스프레드시트 자동 생성)

| 시트명 | 역할 |
|--------|------|
| Data | 검수 대상 데이터 |
| Results | 검수 결과 (키, 식별값1/2, 검수결과, 오매핑유형, 메모, 작업자, 작업시간) |
| Assignments | 작업자별 행 범위 할당 |
| Reasons | 오매핑 사유 목록 및 단축키 |
| Logs | 작업 시간 로그 (작업자별 처리 속도 통계용) |

## 모드

### 표준 모드 (standard)

네이버 ↔ 쿠팡 고정 컬럼 구조. 기존 프로젝트 호환용.

- **좌 그룹 (네이버)**: 상품번호, 상품명, 브랜드명, 네이버_옵션, 네이버_URL
- **우 그룹 (쿠팡)**: C_상품번호, C_상품명, C_옵션명, C_URL
- **필수 컬럼**: 상품번호, 상품명, 네이버_URL, C_상품번호, C_상품명, C_URL

### 자유 모드 (free)

컬럼 구조를 자유롭게 지정. 업로드할 엑셀 형식:

| 행 | 내용 | 예시 |
|----|------|------|
| 1행 | 그룹 헤더 (병합 셀) | `타겟` / `매칭` |
| 2행 | 실제 컬럼 이름 | `상품번호` / `상품명` / `URL` ... |
| 3행~ | 데이터 | — |

> 컬럼명에 `URL`, `link`, `링크` 중 하나가 포함되면 클릭 가능한 링크로 자동 처리된다.

## 기본 오매핑 사유 목록

프로젝트 생성 시 아래 8개 사유가 자동 등록된다. 관리자가 프로젝트별로 추가/삭제/순서 변경 가능.

| 사유명 | 기본 단축키 |
|--------|-------------|
| 다른상품_브랜드 | Q |

## 관리자 가이드

탭 상단 **관리자** 버튼 클릭 → 비밀번호 입력으로 진입.

### 프로젝트 관리

- **새 프로젝트 만들기**: 이름 + 모드(표준/자유) 선택 → 프로젝트 전용 스프레드시트가 `검수툴_프로젝트시트` 폴더에 자동 생성됨
- **이름 변경**: 프로젝트 카드의 "이름 변경" 버튼 → Drive 파일명도 함께 변경됨
- **삭제**: 프로젝트 이름을 정확히 입력 + 관리자 비밀번호 2단계 확인 후 삭제 (되돌릴 수 없음)

### 데이터 업로드

- 지원 형식: `.xlsx`, `.xls`, `.csv`
- **업로드 모드 선택** (기본값: 추가):
  - **추가**: 기존 Data 뒤에 이어 붙임. 기존 Results 키 유지 (검수 결과 보존)
  - **덮어쓰기**: 클릭 시 팝업으로 Results 처리 방식 선택
    - "검수 결과도 함께 초기화" → Data 교체 + Results 삭제 (Logs 유지)
    - "데이터만 교체 (결과 유지)" → Data 교체, Results/Logs 유지
- 자유 모드 추가 시 컬럼 구조가 다르면 업로드 거부 + 에러 표시
- 자유 모드: 1행 병합 정보를 자동 파싱하여 좌/우 그룹 스키마를 결정하고 Projects 시트에 저장

### 작업자 풀 관리

- Workers 시트에 저장되며 모든 프로젝트가 공유
- 작업자 추가/삭제 시 즉시 반영
- 할당된 작업자를 삭제하면 경고 표시됨

### 오매핑 사유 관리

- 프로젝트별 독립
- 사유명 수정 시 Results 시트의 기존 검수 결과 사유도 일괄 업데이트됨
- 단축키: 영문/숫자/한글 단일 글자 (`1`, `2`, `3`, `4`, `←`, `→`는 예약되어 사용 불가)
- ↑↓ 버튼으로 순서 변경 가능

### 행 범위 할당

- 작업자별로 검수할 행 범위(시작~끝)를 지정
- **균등 분배**: 등록된 작업자에게 전체 행을 자동으로 균등 분배
- 1명에게 여러 범위를 중복 할당 가능
- 진행 현황 테이블에서 실시간 완료/할당/진행률 확인 가능

### 통계 및 다운로드

- **검수 결과 통계**: 전체/완료/일치/불일치 건수, 일치율 바 차트, 불일치 사유 분포
  - **날짜 필터**: 전체 기간 / 날짜 선택 / 기간 설정 — 통계 수치 + 작업자별 성과 모두 적용
- **작업자별 성과**: 처리 건수, 일치율, 중간 작업시간(초), 시간당 처리량
- **엑셀 다운로드**: 아래 5개 시트가 포함된 xlsx 파일 생성

| 시트명 | 내용 |
|--------|------|
| 검수결과 | 데이터 + 검수결과 병합 (행번호, 모든 컬럼, 결과/사유/메모/작업자/시간) |
| 전체 진행 현황 | 작업자별 범위/완료/할당/진행률 |
| 검수 결과 통계 | 전체 수치 + 불일치 사유 분포 |
| 작업자별 성과 | 처리 건수, 일치율, 중간 작업시간, 시간당 처리량 |
| 오매핑 사유 목록 | 현재 등록된 사유 및 단축키 |

- **결과 초기화**: Results 시트 + Logs 시트 전체 삭제 (되돌릴 수 없음)

## 작업자 가이드

### 시작 전 설정

1. 상단 **작업자** 탭 선택
2. **프로젝트** 드롭다운에서 작업할 프로젝트 선택
3. **작업자 ID** 입력란에 본인 이름 입력 (자동완성 지원)

### 검수 방법

- 화면 좌/우 카드에 두 상품의 정보가 나란히 표시됨
- **일치** 또는 **불일치** 버튼 클릭 → 즉시 자동 저장
- 불일치 선택 시 오매핑 유형을 추가로 선택
- 메모 입력 후 다음 항목으로 이동 시 자동 저장

### 키보드 단축키

| 키 | 동작 |
|----|------|
| `1` | 일치 |
| `2` | 불일치 |
| `3` | 왼쪽 그룹 URL 열기 |
| `4` | 오른쪽 그룹 URL 열기 |
| `←` / `→` | 이전 / 다음 항목 |
| `Ctrl + Enter` (메모 입력 중) | 다음 항목으로 이동 |
| 사유 단축키 (Q, W, E, R, ...) | 해당 오매핑 유형 선택 및 저장 |

### 내비게이션

| 버튼 | 동작 |
|------|------|
| 이전 / 다음 | 한 항목씩 이동 |
| 이전 미작업 / 다음 미작업 | 미완료 항목으로 바로 이동 |
| 행 번호 입력 + 이동 | 특정 행으로 직접 이동 (내 할당 범위 내) |
| 전체 보기 | 내 할당 전체 목록 팝업 (검색 가능) |

### 상태 표시

| 표시 | 의미 |
|------|------|
| 미작업 | 아직 검수되지 않은 항목 |
| 완료 (일치/불일치) | 본인이 검수한 항목 |
| 완료 · [이름]이 검수함 | 다른 작업자가 이미 검수한 항목 (수정 가능) |

> 결정을 잘못 선택한 경우 **결정 취소** 버튼으로 미작업 상태로 되돌릴 수 있다.

## 동기화

- 검수 결과는 결정/사유 선택 즉시 서버에 저장됨
- 30초마다 서버 상태를 자동 갱신하여 다른 작업자의 변경사항 반영
- 로컬 저장 우선 구간(저장 후 30초)이 있어 동시 저장 충돌을 최소화
- 마지막 선택한 프로젝트, 작업자 ID, 현재 행 위치는 브라우저 `localStorage`에 저장되어 새로고침 후 복원됨
