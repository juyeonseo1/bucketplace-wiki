# QA 및 고객사 체감 정확도 산출

> **작성일**: 2026-05-11
> **상태**: 논의 중 — 아래 내용은 확정 전 초안이며 변경될 수 있음

---

## 개요

버킷플레이스 프로젝트의 매칭 정확도를 정량적으로 측정하기 위한 QA 프로세스 및 정확도 산출 방식.

---

## 데일리 QA 구조

- 총 **100개** QA (작업자 50개 + 시스템 50개)
- 각 50개는 **4번 25개 + 1~3번 25개**로 구성

| 구분 | 개수 | TRUE 판정 기준 |
|------|------|----------------|
| **4번** | 25개 | 매칭 TRUE의 개수를 COUNT |
| **1~3번** | 25개 | 정답이 오늘의집인 건 중 다른 매칭 후보가 없으면 TRUE |

---

## 정확도 산출 계산식

```
1~3번 정확도 = 1~3번 TRUE 개수 / 50
4번 정확도   = 4번 TRUE 개수 / 50

보정 정확도  = (1~3번 정확도 × 모집단 내 1~3번 개수
              + 4번 정확도 × 모집단 내 4번 개수)
              / 모집단 전체 개수
```

> 실제 모집단에서 4번과 1~3번의 비율이 다르므로 모집단 비율 기준으로 보정

---

## 관리자 검수 프로세스 (예정)

- **매일 12:50 / 21:30** 관리자 검수
- 작업자가 검수한 건에 대해 관리자가 수정 여부 + 수정 내용 확인
- 관리자가 뒤집은 건수를 작업자별로 누적 관리

### 작업 부진 작업자 대응

- 작업 timestamp 기반으로 평균 소요 시간 대비 지나치게 빠른 작업자(n초 내 처리) 주의 및 경고
