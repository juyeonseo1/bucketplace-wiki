# 1. 작업 가이드

## 1-1. 주요 작업

타겟 상품(오늘의집 상품)에 대해 아래 내용을 확인하는 작업입니다.

- 타겟 상품이 **시장 최저가인지 여부 확인**
- 타겟 상품보다 **더 저렴한 최저가가 존재하는지 탐색**
- 최저가 정보의 **정확성 검증 및 수정 입력**

## 1-2. 용어 설명

| 용어 | 설명 |
|------|------|
| **타겟 상품** | 최저가 여부를 비교해야 하는 기준 상품 (옵션 단위까지 동일해야 함) |
| **번호** | 할당된 작업 번호 |
| **상품 ID** | 오늘의집 고유 상품 ID |
| **수정 여부** | 검수 전 / 검수 완료 |
| **diff** | 시스템 최저가 vs 타겟가 퍼센트 차이 |
| **수집 갯수** | 시스템이 자동 수집한 가격 정보 개수 |
| **최저가 URL** | 수집 또는 직접 검수한 최저가 상품 URL |
| **매칭 플랫폼** | 최저가 URL 소속 플랫폼명 (영문으로 입력) |
| **상품명** | 최저가 상품명 |
| **상품원가** | 정가 (줄 그어진 가격, 숫자만 입력) |
| **할인가** | 실제 구매 가능한 최저가 (숫자만 입력) |
| **배송비** | 배송비 (숫자만 입력) |
| **배송기준** | 무료배송 기준 금액 (숫자만 입력) |
| **포인트** | *네이버 전용* 구매적립 포인트 |

## 1-3. 수집 갯수 기준 작업 방식

| 수집 갯수 | 작업 방법 |
|----------|-----------|
| **0개** | 작업 불필요 |
| **1개** | 네이버 가격비교에서 브랜드 + 상품명으로 검색 필요 |
| **5개 이하** | 네이버 가격비교 검색 필요 |
| **10개 이상** | 후보 리스트 내에서 정답 탐색 |
| **diff 값 이상** | 시스템 오류 가능성 높음 → 직접 검수 |

## 1-4. 최저가 선정 핵심 원칙

- 옵션까지 완전히 동일해야 동일 상품 (해외상품·리퍼·사은품 제외)
- PDP(상세페이지) 기준 금액 우선
- 네이버와 타 스토어 **가격 동일** → **네이버를 최저가로 선택**
- 네이버와 타 스토어 **가격 다름** → 적립금 제외한 실 판매가 기준으로 가장 낮은 곳 선택
- 네이버 적립금은 최저가 판단 기준에서 제외

## 1-5. 가격 비교 기준

- 오늘의집 vs 타사이트 가격 다름 → **오늘의집 기준 우선**
- PDP(상세페이지) 금액이 시스템 금액과 다르면 → **PDP 기준**
- 두 곳 모두 품절 → **오늘의집 선택**
- 타사이트가 더 저렴 → **타사이트 선택 + 플랫폼명 입력**
- 옵션별 가격 차이 있음 → 옵션 선택 후 실제 구매 금액 기준 입력

## 1-6. 배송 관련

- 배송비·배송기준은 **숫자**만 입력
- 기준: **수도권 배송 기준만 반영**
- 지방 배송비는 제외
- 배송 가능 조건은 고려하지 않고 기재된 배송비로 적용

## 1-7. 기타 규칙

- **장바구니 쿠폰 제외** / 상세페이지(PDP) 기준 금액으로 비교
- 작업일 기준으로 당일만(하루만) 사용 가능한 쿠폰은 적용 대상에서 제외
- 판매원/수입원이 동일하면 **동일 상품**으로 간주
- 오늘의집이 국내상품 → 해외상품 **비교 제외**
- 오늘의집도 해외상품 → 해외 기준 비교 가능
- 검수 완료 상태여도 실제 최저가와 다르면 검수 재확인
- 네이버 '나의 할인가' 적용 항목: 알림 쿠폰 / 제외 항목: 웰컴쿠폰·라운지 고객 쿠폰·최초 할인 쿠폰
- 오늘의집 결제 시 최종 결제 금액이 다를 경우 **최종 결제 금액 입력**
- 오늘의집 상품이 2개 이상인 경우 → 오늘의집이 최저가여도 2순위 최저가 상품을 선택하여 최저가 수정하기 버튼 사용

# 3. FAQ — 자주 묻는 질문

## 동일 상품 기준

- 옵션까지 완전히 동일해야 동일 상품
- 해외상품 / 리퍼 / 사은품 구성은 동일 상품 아님
- 판매원/수입원이 동일하면 동일 상품으로 간주 (브랜드명 다른 경우 포함)

## 수집 갯수별 작업 방법

| 갯수 | 방법 |
|------|------|
| 0개 | 작업 불필요 |
| 1개 | 네이버 가격비교에서 브랜드+상품명 검색 |
| 5개 이하 | 네이버 가격비교 검색 필요 |
| 10개 이상 | 후보 리스트 내에서 정답 탐색 |
| diff 과도하게 크거나 작음 | 시스템 오류 가능성 높음, 직접 검수 |

## 가격 비교 기준

- 오늘의집 vs 타사이트 가격 다름 → **오늘의집 기준 우선**
- PDP(상세페이지) 금액이 시스템 금액과 다르면 → **PDP 기준**
- 두 곳 모두 품절 → **오늘의집 선택**
- 타사이트가 더 저렴 → **타사이트 선택 + 플랫폼명 입력**
- 옵션별 가격 차이 → 옵션 선택 후 실제 구매 금액 기준 입력

## 배송 관련

- 배송비·배송기준은 숫자만 입력
- 수도권 배송 기준만 반영 (지방 배송비 제외)
- 배송 가능 조건은 고려하지 않고 기재된 배송비로 적용

## 최저가 선정 기준

- 네이버와 타 스토어 **가격 동일** → **네이버 선택**
- 네이버와 타 스토어 **가격 다름** → 적립금 제외한 실 판매가 기준 가장 낮은 곳 선택
- 네이버 적립금은 최저가 판단 기준에서 제외

## 네이버 '나의 할인가' 입력 기준

- **적용 항목:** 알림 쿠폰
- **제외 항목:** 웰컴쿠폰 / 라운지 고객 쿠폰 / 최초 할인 쿠폰 등

## 장바구니 / 결제창 할인

- **장바구니 쿠폰 제외**
- **상세페이지(PDP)** 기준 금액으로 비교
- 작업일 기준으로 당일만 사용 가능한 쿠폰은 적용 대상에서 제외
- 오늘의집 결제 시 추가 할인 존재 → 최종 결제 금액 입력

## 해외상품 비교 기준

- 오늘의집이 국내상품 → 해외상품 **비교 제외**
- 오늘의집도 해외상품 → 해외 기준 비교 가능

## 검수 관련

- 검수 완료 상태여도 실제 최저가와 다르면 검수 재확인
- 순번과 관계없이 **실제 최저가 기준**으로 판단

## Retool 데이터 불일치

- 좌측(작업대상)과 우측(후보 리스트) 조회 기준 다름 → **정상 현상**

## 검색 팁

- 네이버 검색 불가 → **구글 검색 활용**
- diff 값 이상하면 → 시스템 오류 가능성 높음, 직접 검수
- 오늘의집 상품이 2개 이상 → 오늘의집이 최저가여도 2순위 최저가 상품을 선택하여 최저가 수정하기 버튼 사용

# 6. 상품 매칭 검수툴 (GAS 기반)

Google Apps Script(GAS) 기반의 웹 검수 툴. 두 플랫폼 간 상품 매칭의 정확성(일치/불일치)을 다수의 작업자가 분업하여 검수하고, 관리자가 진행 현황을 모니터링하고 결과를 다운로드할 수 있다.

## 기술 구조

| 항목 | 내용 |
|------|------|
| 런타임 | Google Apps Script (GAS) |
| 프런트엔드 | 단일 HTML 파일 (index.html) — GAS `doGet()`으로 서빙 |
| 데이터 저장 | Google Spreadsheet (시트별 역할 분리) |
| 파일 파싱 | SheetJS (xlsx 라이브러리, CDN) |

## 기본 오매핑 사유 목록

| 사유명 | 기본 단축키 |
|--------|-------------|
| 다른상품_브랜드 | Q |
| 다른상품_속성상이 (색상, 수량, 사이즈) | W |
| 다른상품_구성상이 (세트 등) | E |
| 다른상품_모델상이 (제목 내 모델명 기준) | R |
| 다른상품_원산지/생산지상이 (식품 등) | A |
| 다른상품_전혀다름 | S |
| 기타_애매한케이스 | D |
| 확인불가(대상/매칭 상품 정보 알 수 없음) | F |

## 작업자 키보드 단축키

| 키 | 동작 |
|----|------|
| `1` | 일치 |
| `2` | 불일치 |
| `3` | 왼쪽 그룹 URL 열기 |
| `4` | 오른쪽 그룹 URL 열기 |
| `←` / `→` | 이전 / 다음 항목 |
| `Ctrl + Enter` (메모 입력 중) | 다음 항목으로 이동 |
| 사유 단축키 (Q, W, E, R, ...) | 해당 오매핑 유형 선택 및 저장 |

## 상태 표시

| 표시 | 의미 |
|------|------|
| 미작업 | 아직 검수되지 않은 항목 |
| 완료 (일치/불일치) | 본인이 검수한 항목 |
| 완료 · [이름]이 검수함 | 다른 작업자가 이미 검수한 항목 (수정 가능) |

> 결정을 잘못 선택한 경우 **결정 취소** 버튼으로 미작업 상태로 되돌릴 수 있다.

# 7. QA 및 정확도 산출

## 데일리 QA 구조

- 총 **100개** QA (작업자 50개 + 시스템 50개)
- 각 50개는 **4번 25개 + 1~3번 25개**로 구성

| 구분 | 개수 | TRUE 판정 기준 |
|------|------|----------------|
| **4번** | 25개 | 매칭 TRUE의 개수를 COUNT |
| **1~3번** | 25개 | 정답이 오늘의집인 건 중 다른 매칭 후보가 없으면 TRUE |

## 정확도 산출 계산식

```
1~3번 정확도 = 1~3번 TRUE 개수 / 50
4번 정확도   = 4번 TRUE 개수 / 50

보정 정확도  = (1~3번 정확도 × 모집단 내 1~3번 개수
              + 4번 정확도 × 모집단 내 4번 개수)
              / 모집단 전체 개수
```

## 관리자 검수 프로세스

- **매일 12:50 / 21:30** 관리자 검수
- 작업자가 검수한 건에 대해 관리자가 수정 여부 + 수정 내용 확인
- 관리자가 뒤집은 건수를 작업자별로 누적 관리
- 작업 timestamp 기반으로 평균 소요 시간 대비 지나치게 빠른 작업자 주의 및 경고
