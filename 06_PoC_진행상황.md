# 버킷플레이스 상품검수 PoC 진행상황

## PoC 개요

- **목표:** 16개 상품 검수 룰을 LLM으로 자동화
- **휴먼 역할:** 시스템 개선 피드백 구축을 위한 지속적 오류 케이스 발굴
- **가이드 사이트:** [상품 검수 가이드](https://juyeonseo1.github.io/product_inspection_guide/)

---

## 1차 작업 (3/10 ~ 3/26)

- **작업 내용:** AI 판단과 휴먼 라벨이 불일치하는 약 1,000건 검수
  - 의료기기 오인 문구, 카테고리 오등록, 자재 등급 관련 위반사항(친환경 문구) 등 우선순위 처리 완료 (817건)
  - F상품(거절 상품) 190건 추가 검수
- **작업 규모:** 총 7명 참여, 3/23~3/26 진행
  - 총 용역 비용: **774,000원**
- **결과:** DS 관리자가 최종 검수 후 클라이언트 전달 완료

---

## 2차 작업 (3/31 ~ 4/21)

- **일정:** 매주 화요일 1회 (3/31, 4/7, 4/14), 오후 2시~7시
- **작업 내용:** 매주 상품 100개씩 정답지 작업 → 시스템 로직 개선에 기여
- **인원:** 기존 참여 인원 동일 (3/31 1회차 완료 — 5명, 14~17시)
- **운영 개선 사항:**
  - 스프레드시트 → **어드민 작업 툴**로 관리 전환
  - 가이드 문서: 스프레드시트 → **Claude 기반 동적 웹 가이드**로 전환
  - 4/14, 4/21 작업 취소

---

## 작업 방향 변경 (4/2 Sync 기준)

- **기존:** 신규 수집 상품 검수 진행
- **변경:** 거절(Fail) 상품 정답 정확도 개선으로 우선 전환
  - 버킷 측 피드백: 승인 세트보다 **거절 상품에서 오류 탐지가 더 중요**
- **4/7 계획:** Pass로 분류된 F상품 약 50~60건 선별 검수 (룰 기준 약 500~600개 규모)
- **휴먼 판단 오류 현황:** 전체 18,000건 룰 중 약 180건(1%)에서 AI 수정 제안 확인됨
  - 어드민 사이트(Azure VPN 발급 후 접근)에서 관리 중

## AgentOS 연동 계획 (주연)

- DS 내부 프로젝트 작업 가이드를 온톨로지 지식사전과 연동
- 온톨로지 챗을 통해 내부 구성원 누구나 가이드 기반 질의응답 가능
  - 예: "버킷플레이스 작업자 근무 시간은?", "최저가 수집 시 포인트는 제외 대상인가?"
- GitHub 및 Firebase를 활용하여 가이드 업데이트 시 HTML 단위로 자동 반영 구조 설계 중
- 작업자가 AgentOS에 직접 접근할 수 없으므로 슬랙에서 @agentos 태그로 질문 → 온톨로지 챗 연동 답변 방식 검토 중
